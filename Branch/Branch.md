# Branch 

### Master
é a ramiicação principal.
quando você faz o commit e você faz um *push* ele redireciona a versão para a **Origin**
você pode fazer commit sem o push.
não é necesário realizar muitos commits na *master*

### Origin
repositório Remoto no Github, uma versão

### Branch
você pode ir commitando pequenas e constantes mudanças e testes no software, quando tudo isso tiver organizado
você realiza um **merge** para juntar uma branch ordinária na master

é possível fazer uma branch sem o merge, caso faça algo sem sucesso, apenas elimine-a
